线程

线程是一个文件（进程）里面有不同的执行路径（多路径执行）。其中的路径是所谓的线程。一般 main 方法叫程序的主线程。

特点：

  • Java 的线程是通过 Java.lang.Tread 类来实现。
  • VM 启动时会有一个由主方法所定义的线程。
  • 可以通过创建 Thread 的实例来创建新的线程。
  • 每个线程都是通过某个特定 Thread 对象所对应的方法 run() 来完成其操作的，方法 run() 称为线程体。
  • 通过调用 Thread 类的 start() 方法来启动一个线程。

线程的优先级

Java 会提供一个线程调度器来监控程序中启动后进入就绪状态的所有线程。
线程调度器按照线程的优先级决定应该调度哪个线程来执行。

  • 线程的优先级用数字来表示，范围从1到10。
  • 一个线程的缺省优先级为5。
  • 优先级的值越大，优先级越高。
  • 通过 setpriortiy 的方式来进行优先级的设置。

Example:

Code

package Lesson_14;

public class TestPriority {

    public static void main(String[] args) {
        // TODO Auto-generated method stub
        Runnable r1 = new T1();
        Runnable r2 = new T2();
        Thread t1 = new Thread(r1);
        Thread t2 = new Thread(r2);
        t1.setPriority(Thread.MAX_PRIORITY);
        t1.start();
        t2.start();
    }

}

class T1 implements Runnable {

    @Override
    public void run() {
        // TODO Auto-generated method stub
        for (int i = 0; i <= 300; i++) {
            System.out.println("T1: " + i);
        }
    }

}

class T2 implements Runnable {

    @Override
    public void run() {
        // TODO Auto-generated method stub
        for (int i = 0; i <= 300; i++) {
            System.out.println("------T2: " + i);
        }
    }

}

线程同步

当有一个线程在对内存进行操作时，其他线程都不可以对这个内存地址进行操作，直到该线程完成操作， 其他线程才能对该内存地址进行操作，这就是线程同步。

Synchronize 锁定（并发执行）

首先我们来看一下，线程之间的同步（相当于进程中的并发）引发的一些特殊的问题。

Example

Code:

package Lesson_14;

public class TestSyn implements Runnable {
    Timer timer = new Timer();

    public static void main(String[] args) {
        // TODO Auto-generated method stub
        TestSyn test = new TestSyn();
        Thread t1 = new Thread(test);
        Thread t2 = new Thread(test);
        t1.setName("t1");
        t2.setName("t2");
        t1.start();
        t2.start();

    }

    public void run() {
        timer.add(Thread.currentThread().getName());
    }

}

class Timer {
    private static int num = 0;

    public void add(String name) {
        num++;
        try {
            Thread.sleep(1);
        } catch (InterruptedException e) {
            // TODO: handle exception
        }
        System.out.println(name + ", 你是第" + num + "个使用timer的线程。");
    }
}

Result:

t1, 你是第2个使用timer的线程。
t2, 你是第2个使用timer的线程。

在上面的例子中我们可以大概的看到，如果当线程一进行 num ++ 操作后执行中断，再由线程二来执行时，num 会再次加一，这样使得线程一和线程二的输出都是“二”。

为了解决这个问题，我们需要用到关键字 Synchronize 来对某一段代码进行对象的锁定。如果有另外一个对象也需要访问这一段代码，则需要等待。

Code:

package Lesson_14;

public class TestSyn implements Runnable {
    Timer timer = new Timer();

    public static void main(String[] args) {
        // TODO Auto-generated method stub
        TestSyn test = new TestSyn();
        Thread t1 = new Thread(test);
        Thread t2 = new Thread(test);
        t1.setName("t1");
        t2.setName("t2");
        t1.start();
        t2.start();

    }

    public void run() {
        timer.add(Thread.currentThread().getName());
    }

}

class Timer {
    private static int num = 0;

    public synchronized  void add(String name) {
        num++;
        try {
            Thread.sleep(1);
        } catch (InterruptedException e) {
            // TODO: handle exception
        }
        System.out.println(name + ", 你是第" + num + "个使用timer的线程。");
    }
}

Result:

t1, 你是第1个使用timer的线程。
t2, 你是第2个使用timer的线程。
