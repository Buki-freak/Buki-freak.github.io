类 与 对象

类用于确定对象将会拥有的特征（属性）和行为（方法）。面向对象编程的核心是：定义好类，然后将类实例化为对象，给一个环境驱使一下，让各个对象之间协作起来形成一个系统。

特点

  • 类是对象的类型。
  • 类是具有相同属性和方法的一组对象的集合。
  • 属性是对象的特定项。
  • 方法是对象执行的操作。
  • 类是抽象的概念，对象是一个确定的实体。
  • 所有的 JAVA 程序都以类（Class）为组织单元。
  • 每个类都可以编写主方法，但是一般情况下，一个系统只允许一个入口，即主方法只有一个。
  • 一个类可以映射多个具有相同属性和方法的对象，即一个类可以对应多个对象。

类的实现

定义类的步骤:

  • 定义类名（类名的首字母大写）
  • 编写类的属性
  • 编写类的方法（可以没有方法）

使用类的步骤

创建对象（实例化）

先了解下 JVM（JAVA 虚拟机）的内存空间，它主要由三部分组成：

  • 方法区内存（只有一处）
  • 堆内存（只有一处）
  • 栈内存（数量与线程有关，一个线程一个栈内存）

三块内存中变化最频繁的是栈内存，最先有数据的是方法区内存，垃圾回收主要针对的是堆内存。

创建语句：
类名 对象名 = new 类名( );
如：People joe = new People( );

创建过程如图：

  • 创建对象的过程中，程序会对电脑类进行实例化成为一个对象，在 JVM（JAVA 虚拟机） 堆内存当中开辟新的内存空间。

  • 实际上这里的 类名( ) 是构造方法，构造方法返回引用数据类型结果给 对象名变量。

  • 对象会涉及到 JAVA 虚拟机内的三个内存区域：方法区内存，栈内存和堆内存。

  • 方法区内存：在类加载的时候，类（Class）字节码代码片段会被加载分配到该内存空间中。静态域也在方法区内存中。

  • 栈内存（存放局部变量）：对象中方法代码片段执行的时候，会给方法分配内存空间，进行压栈操作。方法执行完成之后，其对应的内存空间释放，进行弹栈操作。 对象（在堆内存中）的首地址在 new 的时候赋值给对象名变量，对象名变量存放在栈内存中（内部类除外。内部类的引用为引用数据类型，可以作为成员变量）。由于对象名变量存储对象的首地址，我们称其为引用。而在堆内存中开辟对应的内存空间称为对象。

  • 堆内存：new 对象的成员变量的实例变量在堆内存中存储。

对象属性和方法的使用

引用对象的属性：joe.height = 5;
引用对象的方法：joe.walk();

成员变量和局部变量（MVs and LVs）

  • 成员变量：在类中定义，用来描述对象将要有什么。成员变量包括类变量（静态变量）和实例变量。类变量储存在类的静态域（在方法区内存中），实例变量储存在堆内存对应的对象内存中。
  • 局部变量：在类中的方法定义，在方法中临时保存数据，储存在栈内存中。

两者的区别：

  • 成员变量的作用域在整个类内部都是可见的；而局部变量的作用域仅仅限于定义它的方法。
  • JAVA 会给成员变量一个初始值（byte/short/long/int 类型为 0 ，float/double 类型为 0.0，boolean 类型为 false，char 类型为 \u0000， 引用数据类型为 NULL）；而不会给局部变量赋予初始值。
  • 在方法内部，出现方法内局部变量和类的成员变量同名时，局部变量具有更高的优先级。（考虑使用 this 的情况）
  • 成员变量的实例变量储存在堆内存中；局部变量储存在栈内存中。

构造方法

构造方法属于方法，是定义在 JAVA 类中的一个用来创建并且初始化对象的方法。它与类同名，并且没有 return 语句。每个构造方法实际上执行之后都有一个返回值，值的类型为类对应的引用数据类型。

特点：

  • 如果定义类的时候没有写入构造方法，编译时JAVA 会自动补充一个无参的构造方法，称为缺省构造函数。
  • 如果定义类的时候写入了构造方法，则写入的无论是有参、无参的构造方法，都不会自动补充构造方法。
  • 有参的构造方法可以和无参的构造方法并存。（参考函数重载）
  • 构造方法给属性赋值，并且可以设置值的条件。

静态变量和静态方法（Static）

静态变量属于成员变量，静态方法属于成员方法，但它们都是类级别的变量，和具体的类无关，可以在对象之间达到共享。注意 Main 方法也属于静态方法。

  • 对于属性，若所有的对象都有这个属性，并且所有对象的属性的值是一样的，则建议使用 static 以节省内存开销。

  • 对于方法，若所有的对象都需要这个方法，那么也建议使用 static 以节省内存开销。

  • 静态变量和静态方法在类加载的时候进行初始化，不需要通过创建对象，内存在开辟了。它存储在类的静态域（静态域在方法区内存中）。

  • 有 static 关键字修饰的方法和属性调用格式：类名.属性/方法名（实参），引用的访问方式也可以，但是开发环境编辑器会报错；无 static 关键字修饰的方法和属性调用格式：引用.属性/方法名（实参），并且前提是类有创建了的实例对象。

  • 静态方法中不能访问动态变量和方法。动态方法中可以访问动态变量和方法。

  • 大多数“工具”类的方式是静态方法，因为“工具类”是为了方便编程。

Main 方法

主方法的写法：public static void main(String [] args)（唯一，其他的名为 main 的方法重载）
其中：

  • 主方法默认并且优先执行。
  • public 表示公开的，在任何位置都可以访问。
  • static 表示静态的，不需要创建对象即可访问。
  • void 表示 main 方法执行结束之后不返回任何值。
  • （String [] args）是 main 方法的形式参数列表。

代码块

代码块分别静态代码块和实例代码块。

静态代码块

  • static 可以用于静态代码块。在一个类中可以编写多个静态代码块。
  • 静态代码块在类加载的时候会按照从上到下的顺序执行，并且只执行一次。
  • 在静态代码块会完成预备工作，先完成数据的准备 ，如：初始化连接池，解析 XML 配置文件。
  • 静态代码块是 JAVA 为程序员准备的一个特殊时刻，这个特殊时刻被称为类加载时刻。
  • 静态代码块的执行会先于 Main 方法。

Example:

test.class

public class test {

    static {
        System.out.println("类加载 --> 1.....");
    }

    static {
        System.out.println("类加载 --> 2.....");
    }

    public static void main(String[] args) {
        System.out.println("Main start....");
    }
}

Result

类加载 --> 1.....
类加载 --> 2.....
Main start....

实例代码块（使用少）

  • 实例代码块可以编写多个，也是遵循自上而下的顺序执行。
  • 实例代码块在构造方法执行之前执行，构造方法依次执行，实例代码块对应执行一次。
  • 实例代码块是 JAVA 为程序员准备的一个特殊时机，这个特殊时机被称为对象初始化时机。

this

关于 this 关键字：

  • this 是一种引用数据类型变量，属于对象的引用，this 变量中保存了内存地址指向自身，this 储存在 JAVA 虚拟机堆内存中。
  • 创建 100 个 JAVA 对象，每一个对象都有 this ，也就说有 100 个不同的 this。
  • 由于 this 在不同对象中的值不同，属于一个动态变量，所以在静态方法中不可以使用 this。
  • this 有些情况下可以省略。
  • 在动态方法中，由于局部变量的优先级大于成员变量，如果出现局部变量与成员变量同名，则成员变量前需要使用 this，以避免混淆。
  • 可以使用在构造方法当中，this 语句通过当前的构造方法调用其他的构造方法。（注意 this 语句只能放在构造函数的第一行且只能调用一次）

Example:

apple.class

public class apple {
    private int price;

    private int weight;

    public apple() {
        this(10, 10);  // 在无参数构造方法中调用两参数构造方法
    }

    public apple(int price, int weight) {
        this.price = price;
        this.weight = weight;
    }

    public void print() {
        System.out.println("the price is : " + price);
        System.out.println("the weight is : " + weight);
    }
}

Main Function

public class test {

    public static void main(String[] args) {
        apple a = new apple();
        a.print();
    }
}

Result

the price is : 10
the weight is : 10

super 关键字

super 代指父类、超类，可以用于调用父类中的普通方法和构造方法。

关于 super 方法：

  • 如果 super 没有被显式地调用，编译器会自动地将 super() 作为子类中构造方法的第一条语句。

Example

public ClassName{
    // some statements
}

等价于

public ClassName{
    super();
    // some statements
}
  • 在任何情况下，构造一个类的实例时，将会调用沿着继承链的所有父类的构造方法。

final 关键字

当我们不希望有子类来重写我们所指定的方法或者不希望变量被改变的时候，我们可以使用 final 这个关键字。
以下三种情况：

  • final 的变量的值不能够被改变。（成员变量和局部变量）
  • final 的方法不能够被重写。
  • final 的类不能够被继承。

典型的 final 类：String，Math，Boolean 等等。

继承方法链

当构造一个子类的对象时，子类构造方法会在完成自己的任务之前，首先调用它的父类的构造方法。如果父类继承自其他类，那么父类构造方法又会在完成自己的任务之前，调用它自己的父类的构造方法，这整个机制被称为继承方法链。

垃圾回收机制（GC 机制）

对于一个堆内存的对象，如果后继代码中没有更多的对象指向它或者它无法被访问的时候，这个对象会变为垃圾数据，按照 GC 机制，垃圾回收器会将垃圾进行回收。

机制的具体内容：JAVA GC 机制

封装性

封装的好处

  • 封装之后，对于事物来说，看不到这个事物比较复杂的一面，只能看到这个事物比较简单的一面。【 简单化 】
  • 封装之后才能成为真正地对象，成为真正地独立体。
  • 封装之后可以使这个事物的适应性增强，在任何情景都可以使用。
  • 封装，对于这个事物本身来说提高了安全性。

封装的步骤

  • 所有属性私有化，使用 private 关键字进行修饰，private 表示私有的，其修饰的所有数据只能在本类中访问。
  • 对外提供简单的操作入口，也就是说以后访问 private 属性必须通过这些简单的入口 进行访问。
  • 入口方法处编写业务逻辑代码进行安全控制，达到过滤的效果。
  • 通常的访问类型有 get 和 set，访问的入口方法命名规范为： set/get + 属性名首字母大写。这两种方法没有 static 关键字。（Eclipse 使用 source –> generate setters and getters 自动生成）

Example:

apple.class

// class-->apple
public class apple {
    private int price = 10;

    public void setPrice(int tempPrice){  // set 方法不需要返回值
        if(tempPrice > 15){
            System.out.println("价格过高！");
            return;
        }
        price = tempPrice;
    }

    public int getPrice() {  // get 方法需要返回值
        return price;
    }

}

Main Function

public class test {
    public static void main(String[] args) {

        apple a = new apple();
        a.setPrice(100);
        int price = a.getPrice();
        System.out.println(price);
    }
}

Result

价格过高！

继承性（extends）

继承是从已有的类中派生出新的类，新的类能吸收已有类的数据属性和行为，并能扩展新的能力。

关于继承：

  • 继承的基本作用是：代码复用。
  • 继承的重要作用是：为“方法的覆盖”和 “多态机制”做铺垫。
  • JAVA 当中的继承只支持单继承，一个类不能同时继承很多类，只能继承一个类。（相比于 C++ ）
  • 若在继承中，B 类继承 A 类，则 A 类被称为：父类、基类、超类、superclass； A 类被称为：子类、派生类、subclass。
  • 私有类（private）和构造方法不支持继承（但依然可以通过公有接口访问）。
  • 类与类之间可以间接继承。
  • JAVA 中假设一个类没有显示继承任何类，该类默认继承 JAVA SE 库中提供的 java.lang.object 类。JAVA 中所有类都具有 object 类的特征。

查看底层代码：
java_default_extends_class.png

这里提到这个 object 类是整个类层次结构的根类，每个类默认都以这个类为超类。

方法覆盖（override）

当父类中的方法已经无法满足子类的业务需求，子类有必要将父类中继承过来的方法进行重新编写，这个重新编写的过程叫做方法覆盖或者方法重写。

关于方法覆盖：

  • 方法覆盖发生在具有继承关系的父子类之间。
  • 方法覆盖的时候，返回值类型相同，方法名相同，参数列表相同。
  • 方法覆盖的时候建议 copy 之后再重写，避免出错。（方法名大小写敏感）
  • 访问权限不能更低，只能更高。权限从高到低依次是：public -> protected -> private，例如 private 可以重写为 public，但是 public 不可以重写为 private。
  • 抛出异常不能更多只能更少。
  • 存在三种方法不能够进行方法覆盖：私有方法、构造方法、静态方法。

Example:
fruit.class

public class fruit {
    public void rid() {
        System.out.println("We get rid of fruit!");
    }
}

apple.class

public class apple extends fruit{
    // Not to Override the Function
}

peer.class

public class peer extends fruit{
    public void rid() {
        System.out.println("We get rid of peer!"); // Override the Function
    }
}

Main Function

public class test {

    public static void main(String[] args) {

        apple a = new apple();
        a.rid();

        peer p = new peer();
        p.rid();

    }
}

Result

We get rid of fruit!
We get rid of peer!

多态性

同一操作作用于不同的对象，可以有不同的解释，产生不同的执行结果就是用基类的引用指向子类的对象。

关于多态：

  • 向上转型（upcasting）:子类型转为父类型，属于自动类型转换。向上转型较为常见。
  • 向下转型（downcasting），父类型转为子类型，属于强制类型转换（需要加强制类型转换符号）。
  • JAVA 程序分为编译阶段和运行阶段。

静态绑定和动态绑定

代码真正运行前需要经过两个阶段，编译和运行。
先看一段代码：

fruit.class

public class fruit {
    public void print() {
        System.out.println("fruit is delicious!");
    }
}

apple.class

public class apple extends fruit{
    public void print() {
        System.out.println("apple is delicious!");
    }

    public void print_apple() {
        System.out.println("apple is red");
    }
}

Main Function

public static void main(String[] args) {
    fruit a = new apple(); // upcasting
    a.print();

    apple a1 = (apple) a; // downcasting
    a1.print_apple();
}

Result

apple is delicious!
apple is red

对于向上转型：

  • 只有静态绑定成功之后，即编译通过之后， 才有后续的运行。
  • 在编译阶段，编译器会检查 a 这个引用的数据类型为 fruit，由于fruit.class 的字节码中有对应调用的 print 方法，编译通过了。这个过程我们称之为静态绑定，编译阶段绑定。
  • 在运行阶段，JVM 内存当中真实创建的对象是 apple 的对象，那么以下程序在运行阶段一定会调用 apple 对象的 print 方法。这个过程我们称之为动态绑定，运行阶段绑定。
  • 无论是否具有方法重写，运行阶段一定是调用 apple.class 中的 print 方法，因为底层真实存在的是 apple 的对象。
  • 本质上这是父类型引用指向子类型对象导致程序存在编译阶段绑定和运行阶段绑定两种截然不同的奇特形态，这是典型的多态。

对于向下转型：

  • 当调用的方法是子类型中特有的，在父类型当中不存在的，必须进行向下转型，如例子中的 print_apple。向下转型的过程中可能会出现类型转换异常 ClassCastException。出现该异常的原因是程序存在安全隐患，编译通过了但是运行未能通过。

类型转换异常

解决办法：强制类型转换前先使用 instanceof 进行判断。

Example:
fruit.class

public class fruit {
    public void print() {
        System.out.println("fruit is delicious!");
    }
}

apple.class

public class apple extends fruit{
    public void print() {
        System.out.println("apple is delicious!");
    }

    public void print_apple() {
        System.out.println("apple is red");
    }
}

Main Function

public static void main(String[] args) {
    fruit a = new apple();

    apple a1 = null;
    if (a instanceof apple) {  // 加上判断判断对象的 instanceof 避免向下类型转换异常
         a1 = (apple)a;
    }

    a1.print();
}

多态的作用

  • 降低程序的耦合度，提高程序的扩展力。
  • 能使用多态尽量使用多态。
  • 面向抽象编程，尽量不要面向具体编程。
  • 类函数以父类为参数接口，使用子类继承父类，并使用向上类型转换的方式以提高程序扩展力。

抽象类与抽象方法

用 abstract 关键字来修饰一个类的时候，这个类就叫做抽象类。同理，用 abstract 关键字类修饰一个方法是，这个方法就叫做抽象方法。

抽象类的特点：

  • 含有抽象方法的类必须被声明为抽象类。
  • 抽象类必须被继承。
  • 抽象类中的抽象方法必须要被重写。
  • 抽象类不能被实例化。
  • 抽象方法只需要声明，不需要实现。
  • 抽象类可以继承抽象类，同时抽象方法也会被继承。
  • 构造方法不能被继承，不能被重写，所以构造方法不能是抽象的。

Example:
Animal.class

public abstract class Animal{
    private String name;
    public Animal(String name) {
        // TODO Auto-generated constructor stub
        this.name = name;
    }
    public abstract void enjoy();
}

Cat.class

public class Cat extends Animal{
    private String eyesColor;
    public Cat(String n, String c) {
        // TODO Auto-generated constructor stub
        super(n);
        eyesColor = c;
    }
    public void enjoy() {
        System.out.println("猫叫声...");
    }
}

test.class

public class test {

    public static void main(String[] args) {
        // TODO Auto-generated method stub
            Cat c = new Cat("Lily", "Blue");
            c.enjoy();
    }
}

接口

接口是抽象方法和常量值定义的集合。

接口的特点：

  • 从本质上讲，接口是一种特殊的抽象类，这种抽象类中只包含常量和方法的定义，而没有变量和方法的实现。
  • 接口可以弥补 JAVA 中一个类只能继承一个类的缺点。
  • 接口中的方法全部都是抽象方法（abstract 关键字可缺省），并且这些方法默认为 public 也只能是 public。
  • 属性的关键字必须为 public static final。
  • 接口可以继承其他的接口，并且添加新的属性和新的抽象方法。

Example:

interface Singer {
    public void sing();

    public void sleep();
}

interface Painter {
    public void paint();

    public void eat();
}

class Student implements Singer {
    private String name;

    Student(String name) {
        this.name = name;
    }

    public void study() {
        System.out.println(this.name + ": is studying");
    }

    public String getName() {
        return name;
    }

    public void sing() {
        System.out.println(this.name + ":student is singing");
    }

    public void sleep() {
        System.out.println(this.name + ":student is sleeping");
    }
}

class Teacher implements Singer, Painter {
    private String name;

    public Teacher(String name) {
        this.name = name;
    }

    public void teach() {
        System.out.println(this.name + ":teacher is teaching");

    }

    public void sing() {
        System.out.println(this.name + ": teacher is singing");
    }

    public void sleep() {
        System.out.println(this.name + ":teacher is sleeping");
    }

    public void paint() {
        System.out.println(this.name + ":teacher is painting");
    }

    public void eat() {
        System.out.println(this.name + ":Teacher is eating");
    }
}

public class TestInterface {
    public static void main(String args[]) {
        Student s0 = new Student("le");
        s0.sing();
        s0.sleep();
        s0.study();

        Teacher t0=new Teacher("te");
        t0.sing();
        t0.sleep();
        t0.paint();
        t0.eat();
        t0.teach();

        Singer s1 = new Student("ce");
        s1.sing();
        s1.sleep();


        Singer s2 = new Teacher("le");
        s2.sing();
        s2.sleep();

        Painter p1 = (Painter) s2;
        p1.paint();
        p1.eat();
}
}

从上面的例子中可以看到几个特点：

  • 多个无关的类可以实现同一个接口。
  • 一个类可以实现多个无关的接口。
  • 与继承关系类似，接口与实现类之间存在多态性。（Teacher 既是 Singer 也是 Painter）

IO

分类

Java.io 包中定义了多个流类型（类或者抽象类）来实现输入 / 输出功能；可以从三个角度进行分类：

  • 按数据流方向的不同可以分为输入流和输出流。
  • 按处理数据单位不同可以分为字节流(8 bits)和字符流(16 bits)。
  • 按照功能不同可以分为节点流和处理流。

Java.io 中的类都分别继承自以下四个抽象类：

节点流和处理流的区别：

  • 节点流可以从一个特定的数据源（节点）读取数据（如：文件，内存）。
  • 处理流是“连接”在已存在的流（节点流或处理流）之上，通过对数据的处理为程序提供更为强大的读写功能。

// 读取一个字节并且以整数的形式返回（0 ~ 255），

处理流类型

处理流可以理解为包到别的流上的流。

分类如下：

Buffered 缓冲流

缓冲流需要“套接”在相应的节点流之上，对读写的数据提供了缓冲的功能，提高了读写的效率，同时增加了一些新的方法。

一般有四种缓冲流：

  • BufferedReader

–> BufferedReader(Reader in)
–> BufferedReader(Reader in, int sz) // sz为自定义缓存区的大小

  • BufferedWriter

–> BufferedWriter(Writer out)
–> BufferedWriter(Writer out, int sz)

  • BufferedInputStream

–> BufferedInputStream(InputStream in)
–> BufferedInputStream(InputStream in, int sz)

  • BufferedOutputStream

–> BufferedOutputStream(OutputStream out)
–> BufferedOutputStream(OutputStream out, int sz)
